Snowflake расширила платформу Snowflake Intelligence новыми коннекторами на базе Model Context Protocol. Теперь агенты внутри Snowflake могут обращаться к Gmail, Google Calendar, Jira, Salesforce и Slack без отдельных интеграционных слоёв. Cortex Code дополнительно получил поддержку Agent Communication Protocol.

## Что изменилось

До этого обновления Snowflake Intelligence работал в первую очередь с данными внутри warehouse. Аналитический агент мог писать SQL, строить отчёты и комментировать таблицы, но был отрезан от рабочих коммуникаций. MCP-коннекторы убирают это ограничение.

Перечисленные источники подключаются по стандартной MCP-спецификации. Для пользователя процесс выглядит как авторизация OAuth и выбор scope. Агент после этого может читать письма, создавать события календаря, открывать тикеты в Jira и публиковать сообщения в Slack от имени человека.

## Cortex Code и ACP

Cortex Code — среда для написания и запуска кода внутри Snowflake. Поддержка Agent Communication Protocol позволяет ей общаться с внешними AI-системами по единому контракту. ACP описывает, как агенты разных вендоров обмениваются задачами, результатами и состоянием.

Snowflake позиционирует комбинацию MCP плюс ACP как control plane для агентного предприятия. MCP отвечает за доступ агента к инструментам и данным. ACP — за межсистемное взаимодействие между самими агентами.

## Практические сценарии

Несколько примеров, которые Snowflake называет в анонсе:

- Агент анализирует продажи в warehouse и сам отправляет резюме в Slack.
- Поддержка читает Jira-тикет, проверяет данные клиента и предлагает ответ.
- Менеджер просит агента собрать митинг с тремя коллегами по их календарям.
- Внешний агент партнёра запрашивает у Cortex Code расчёт по ACP-контракту.

## Что это значит

Для команд на Snowflake снижается порог построения мультисистемных агентов. Раньше каждая интеграция требовала отдельного коннектора и слоя аутентификации. Теперь типовые рабочие источники доступны как MCP-сервисы из коробки.

Решение также фиксирует ставку Snowflake на открытые протоколы. Компания предпочла MCP и ACP проприетарным API, что упрощает миграцию и совместимость. Конкуренты Databricks и Microsoft Fabric также продвигают MCP, но конкретный набор коннекторов и поддержка ACP отличаются. Доступность функций объявлена как поэтапный rollout в течение второго квартала 2026 года.
