OpenAI выпустила GPT-5.5 — новую версию флагманской линейки. Модель доступна в двух конфигурациях: GPT-5.5 и GPT-5.5 Pro. Обе работают через API; в ChatGPT обновление раскатывается постепенно по тарифам Plus, Team и Enterprise.

Главные изменения коснулись трёх направлений: кодирование, многошаговые агентные задачи и работа с длинными документами. По заявлениям OpenAI, GPT-5.5 заметно лучше держит контекст в проектах с десятками файлов и реже теряет нить при цепочках инструментов.

На бенчмарке ARC-AGI-2 модель показывает 85% против 73% у GPT-5.4. Прирост связан с обновлённой схемой reasoning: модель тратит больше времени на планирование шагов и реже сваливается в локальные циклы. На SWE-bench Verified прирост скромнее — около 4 процентных пунктов, но стабильный на разных типах задач.

Что важно для разработчиков:

- Контекст расширен до 1 млн токенов в обеих конфигурациях.
- Структурированный вывод поддерживает JSON Schema без обходных промптов.
- Tool use стал детерминированнее: при одинаковом state модель чаще выбирает один и тот же инструмент.
- Появился отдельный режим `agent` с автоматическим резюме промежуточных шагов.
- Стоимость GPT-5.5 — на уровне GPT-5.4; Pro дороже примерно в три раза.

GPT-5.5 Pro отличается удлинённым reasoning-циклом и более высоким качеством на задачах с математикой и доказательствами. На олимпиадных задачах USAMO модель решает 6 из 6 в зачётном времени, тогда как обычная версия — 4 из 6. Для типичных продуктовых сценариев Pro избыточен и дороже без явного выигрыша.

Тонкая настройка пока не открыта. OpenAI обещает supervised fine-tuning через 2–3 недели, RLHF — позже. Vision и аудио работают сразу, голосовой режим в ChatGPT тоже переключился на новую модель.

Из ограничений стоит держать в голове два момента. Первый: на коротких диалогах GPT-5.5 иногда «передумывает» лишний раз и отвечает медленнее GPT-5.4. Для интерактивных интерфейсов это заметно. Второй: тарификация reasoning-токенов изменилась — теперь они считаются отдельной строкой и могут заметно увеличить счёт при включённом параметре `reasoning_effort: high`.

Миграция с GPT-5.4 не требует изменений в коде. Достаточно поменять идентификатор модели на `gpt-5.5` или `gpt-5.5-pro`. OpenAI рекомендует проверить промпты, использующие старый формат function calling — он формально поддерживается, но новый tools API быстрее и стабильнее.

GPT-5.4 остаётся в API ещё минимум 12 месяцев. Дату вывода из эксплуатации OpenAI обещает сообщить заранее.
