Center for Internet Security совместно с Astrix Security и Cequence выпустил companion guide к CIS Controls. Документ посвящён защите Model Context Protocol-серверов и жизненного цикла AI-агентов в корпоративных средах. Это первая попытка отраслевого стандартизатора систематизировать риски агентного стека.

## Что внутри

Руководство фокусируется на трёх уровнях контроля:

- Управление non-human identities, через которые агент получает токены и ключи.
- Ограничение scope MCP-инструментов и явная декларация допустимых действий.
- Аудит tool-вызовов с трассировкой источника инструкции и результата.

Авторы указывают, что MCP-сервер становится новым периметром. Через него агент получает доступ к CRM, репозиториям, тикет-системам и базам данных. Если сервер скомпрометирован или плохо сконфигурирован, эти системы открыты как через единый API.

## Основные риски

В гайде перечислены типовые векторы атак. Среди них prompt injection через возвращаемые tool-результаты, утечка секретов в логи MCP, чрезмерные права у service-account, отсутствие rate limit на чувствительные операции. Отдельный раздел разбирает supply chain — публичные MCP-серверы из реестров без проверки подписи кода.

Документ предлагает маппинг рекомендаций на существующие CIS Controls v8. Это позволяет встроить требования в действующие программы соответствия без отдельного фреймворка. Для каждого контроля указано, к какой стадии жизненного цикла агента он применяется: discovery, deployment, runtime, decommission.

## Практические шаги

Для команд, уже использующих MCP в проде, авторы рекомендуют:

- Инвентаризировать все MCP-серверы и их инструменты.
- Прописать allowlist домена и команд для каждого агента.
- Включить структурное логирование вызовов с корреляцией по сессии.
- Запустить периодический review прав агентов и их владельцев.
- Изолировать чувствительные tool-вызовы за человеческим подтверждением.

Гайд распространяется бесплатно через сайт CIS. Astrix и Cequence выступили техническими соавторами и предоставили примеры из реальных инцидентов у клиентов. Документ не отменяет специфичных требований PCI DSS или HIPAA, но даёт общую рамку для выстраивания governance.

Для разработчиков MCP-инструментов это сигнал. Энтерпрайз-внедрения теперь будут опираться на формальные критерии оценки, и серверы без аудита прав и логирования начнут отсеиваться на этапе закупки.
