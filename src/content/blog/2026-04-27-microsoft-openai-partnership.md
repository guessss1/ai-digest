Microsoft и OpenAI объявили о пересмотре партнёрского договора, действовавшего в прежней редакции с 2023 года. Соглашение продлено до 2032 года, но условия заметно изменились в обе стороны.

Главное изменение касается мультиоблачности. Раньше OpenAI могла обслуживать коммерческих клиентов только через Azure. Теперь это ограничение снято: OpenAI вправе разворачивать инференс и обучать модели на любом облачном провайдере. На практике это означает, что клиенты OpenAI смогут получать сервис через AWS, Google Cloud или Oracle, не используя Azure как обязательную прослойку.

Второе изменение — лицензия Microsoft на технологии OpenAI становится неэксклюзивной. До сих пор у Microsoft были привилегированные права на использование моделей OpenAI в собственных продуктах: Copilot, Bing, Azure OpenAI Service. Эти права сохраняются, но теперь OpenAI может лицензировать те же технологии другим компаниям на сравнимых условиях.

Третье — Microsoft перестаёт получать revenue share с выручки OpenAI. Прежняя схема предусматривала отчисления Microsoft с доходов OpenAI до возврата инвестиций и сверх того. Теперь поток в одну сторону прекращён. Вместо этого Microsoft получит расширенные права на интеллектуальную собственность OpenAI после достижения определённых порогов AGI-капабилитей — конкретные критерии в публичной части не раскрыты.

Что остаётся прежним:

- OpenAI остаётся первичным партнёром Azure для инференса и обучения.
- Microsoft сохраняет место в совете директоров OpenAI.
- Azure OpenAI Service продолжает получать модели OpenAI с приоритетом по времени выпуска.
- Совместные обязательства по безопасности и red-teaming моделей сохранены.

Для корпоративных клиентов изменения практические. Те, кто привязан к AWS или Google Cloud, теперь смогут получать модели OpenAI без построения мостов через Azure. Для существующих клиентов Azure OpenAI Service ничего не меняется — те же модели, те же SLA, те же региональные доступности.

Для OpenAI это шаг к большей операционной независимости. Компания получает свободу выбирать инфраструктурного партнёра под конкретный сегмент клиентов и снимает структурный конфликт интересов с Microsoft на корпоративном рынке. Также упрощается выход на рынки, где Azure исторически слаб — например, государственный сектор в ряде стран.

Для Microsoft потеря revenue share компенсируется снятием регуляторного давления. В США и ЕС антимонопольные органы последний год активно изучали структуру партнёрства; формальная неэксклюзивность снимает значительную часть вопросов. Параллельно Microsoft продолжает развивать собственные модели в рамках MAI-инициативы.

Финансовые детали сделки в публичной версии не раскрыты.
