21 апреля Google добавила в Gemini API двух агентов — Deep Research и Deep Research Max. Раньше Deep Research был фичей приложения Gemini, теперь он доступен программно, а его старшая версия Max — только через API и только на платных тирах. В приложении Gemini варианта Max нет.

По характеристикам: контекст до 1 миллиона токенов на вход, output — до 65 тысяч. Агент умеет синтезировать отчёты из сотен веб-источников и пользовательских файлов за один прогон. Поддерживается MCP (Model Context Protocol) — значит, можно подключать внешние инструменты и источники данных по открытому протоколу. Также работает File Search — встроенный механизм поиска по загруженным документам, без необходимости руками собирать RAG-пайплайн.

Под капотом, по данным Google, Gemini 3.1 Pro. Заявленные метрики: 93.3% на DeepSearchQA и 54.6% на Humanity's Last Exam. Второй бенчмарк — один из самых сложных в индустрии, и 54.6% — заметная цифра для открыто задокументированного агента. Как обычно, это замеры самой компании, и независимые прогоны могут дать другие числа.

Практическая разница между Deep Research и Deep Research Max — в глубине и объёме работы. Max рассчитан на задачи, где нужно долго копать: обзоры литературы, конкурентный анализ, технические due diligence, где агенту приходится ходить по сотням источников, сравнивать, верифицировать и писать структурированный отчёт. Обычный Deep Research — для более лёгких сценариев, где хватает быстрой сводки.

Что это значит для интеграций. Раньше похожие сценарии приходилось собирать самостоятельно: брать Gemini, писать оркестрацию поисковых запросов, борьбу с галлюцинациями, цитирование, резолв конфликтующих источников. Теперь это завёрнуто в один endpoint, который возвращает отчёт с ссылками. Для продуктовых команд, которым нужен research-as-a-feature, — это экономия инженерных усилий и предсказуемое поведение.

Ограничения. Во-первых, только платные тиры API — на free tier Max недоступен. Во-вторых, стоимость такого прогона, судя по объёму работы (сотни источников, 65K output), будет заметной, и Google явно таргетирует корпоративного пользователя, а не индивидуальных разработчиков с грантом. В-третьих, агент — это preview-модель (слово preview прямо в имени роута), так что контракт API может меняться до GA.

Для задач, где важна глубина, а не скорость, Deep Research Max — один из первых API-доступных агентов такого класса от крупного вендора. Для повседневных чатов — избыточен и дорог.
