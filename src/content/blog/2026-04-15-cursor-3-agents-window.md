2 апреля Anysphere выпустила Cursor 3 — первый крупный переосмысленный релиз редактора с тех пор, как они добавили Composer в 2024. Главное изменение — интерфейс. Cursor 3 строится вокруг Agents Window: отдельной панели для запуска и наблюдения за агентами, работающими параллельно. До восьми одновременно, каждый в своём окружении — локально, по SSH, в git worktree или в облаке Anysphere.

Идея простая: когда у задачи нет очевидно правильного решения, дешевле запустить три реализации в параллельных worktree и сравнить, чем выбирать заранее. Для этого сверху добавлена Mission Control — дашборд, показывающий, что каждый агент сейчас делает, какие файлы трогал и какие решения принял. Выглядит как оркестратор, а не как «чат с редактором», и это главный сдвиг.

Вторая часть релиза — Composer 2, собственная frontier-модель Anysphere. По CursorBench — внутренний бенчмарк — она набирает 61.3, что на 39% выше Composer 1.5. Скорость — 200+ токенов в секунду на кастомных GPU-ядрах, которые Anysphere собирает сама. Модель стала default в Auto-режиме, но Claude, GPT-5.4 и Gemini по-прежнему доступны как альтернативы. Цена Composer 2 — $0.50 за миллион входных и $2.50 за миллион выходных токенов, что заметно ниже frontier-моделей у Anthropic и OpenAI.

Контекст: Cursor давно соперничает с Windsurf и Claude Code. У Windsurf есть Arena Mode — похожее сравнение моделей на одной задаче — но без параллельных агентов в разных worktree. Claude Code добавил сабагенты и worktree-изоляцию в Insiders 1.117 VS Code в середине апреля, и линии начинают сходиться: в 2026 все три инструмента решают одну проблему — как дать разработчику не одного помощника, а парк.

Для команд, которые уже работают в Cursor, апдейт бесшовный — тот же редактор, новые панели появились сами. Для тех, кто сидит на VS Code с Copilot или Claude Code, это повод ещё раз сравнить: сценарий «запусти три варианта, выбери лучший» в Cursor 3 делается ощутимо меньшим числом кликов, а Mission Control снимает необходимость помнить, какой агент что делает.
