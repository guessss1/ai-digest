AWS выкатила крупное обновление Amazon Bedrock AgentCore. В одном анонсе три фичи, каждая из которых заметно меняет процесс разработки агентов на платформе.

Первая — managed agent harness. Раньше для запуска агента нужно было вручную собирать связку из IAM-ролей, Lambda-функций, S3-бакета для артефактов и DynamoDB-таблицы для состояния. Теперь это укладывается в три API-вызова: создать агента, прикрепить инструменты, запустить. Под капотом AgentCore сам разворачивает инфраструктуру, выписывает временные креды и поднимает мониторинг.

Вторая — AgentCore CLI. Утилита `agentcore` ставится через `npm` или `pip`, поддерживает локальный запуск агента, дебаг-сессии с пошаговым выполнением и экспорт конфигурации в CDK или Terraform. Локальный запуск использует моки облачных сервисов, так что итерироваться можно без поднятия инфраструктуры. Команда `agentcore deploy` синхронизирует локальное определение с облачным и показывает диф перед применением.

Третья — persistent filesystem. Каждый агент получает выделенную файловую систему, которая переживает перезапуски и паузы. Это закрывает типичный кейс долгих задач: агент обрабатывает датасет несколько часов, упирается в лимит Lambda, AgentCore сохраняет состояние и FS, а после возобновления агент продолжает с того же места. Размер по умолчанию — 10 ГБ, расширяется до 1 ТБ.

Отдельной строкой идёт набор готовых coding skills для популярных coding-агентов:

- Claude Code — интеграция через MCP-сервер, доступ к AWS-ресурсам без отдельной настройки.
- Codex — официальный коннектор для деплоя и логов.
- Cursor — расширение для IDE с автодополнением по схемам сервисов AWS.

Coding skills бесплатны до конца апреля. После — тарификация по запросам к Bedrock плюс стандартная плата за используемые AWS-ресурсы.

Для разработчиков, уже работающих с AgentCore, миграция несложная. Существующие агенты продолжают работать в прежнем режиме. Чтобы перейти на managed harness, нужно переписать определение в новом YAML-формате — в репозитории есть автоконвертер старых конфигураций. Persistent FS подключается флагом `persistentFilesystem: true` без изменений в коде агента.

Из ограничений стоит держать в голове три. Первое: managed harness пока работает только в регионах us-east-1, us-west-2 и eu-west-1. Второе: CLI требует Node.js 20 или Python 3.11 и не работает в Windows нативно — только через WSL. Третье: persistent filesystem не реплицируется между регионами; для multi-region агентов нужно делать снапшоты и синхронизировать вручную.

Документация и примеры доступны в AWS Samples на GitHub.
