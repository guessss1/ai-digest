Google добавила в Gemini API механизм event-driven webhooks. Раньше для длинных задач — Batch-инференс или генерация видео в Veo — приходилось периодически опрашивать статус через polling. Это съедало квоты, добавляло задержки и плохо ложилось на serverless-архитектуры. Теперь разработчик регистрирует HTTP-эндпоинт, и API сам присылает POST-запрос в момент завершения задачи.

Подписка работает в двух режимах:

- Статический — один эндпоинт принимает события всех типов; настраивается один раз в консоли.
- Динамический — URL передаётся в теле запроса при создании задачи; полезно для маршрутизации в разные сервисы.

Каждое уведомление приходит с подписью HMAC-SHA256. Получатель вычисляет хэш от тела запроса с общим секретом и сверяет с заголовком. Это закрывает классический риск webhook-эндпоинтов — приём поддельных событий от стороннего источника. Секрет генерируется при создании подписки и больше не показывается, ротация — через консоль.

Если эндпоинт временно недоступен, API повторяет доставку в течение 24 часов с экспоненциальным бэкоффом. Это снимает с разработчика обязанность держать буфер на случай деплоев и кратковременных сбоев. После 24 часов событие отбрасывается, статус задачи остаётся доступен через GET-запрос.

Поддерживаемые на старте сценарии:

- Batch API — массовые инференс-запросы к моделям Gemini Pro и Flash.
- Veo — генерация видео, типичное время выполнения от минут до часов.
- File Search — переиндексация больших корпусов после загрузки.

Тело уведомления содержит идентификатор задачи, статус, ссылку на результат и метаданные. Для Veo это URL готового видео и кадры превью. Для Batch — путь к JSONL-файлу с результатами на Google Cloud Storage. Разработчик отвечает HTTP 2xx и забирает результат отдельным запросом.

Webhooks устраняют типичную проблему serverless-функций: при polling-схеме нужна была долгоживущая worker-задача или cron, которые периодически дергают API. Теперь функция стартует только когда событие реально пришло. Это удешевляет интеграцию для тех, кто строит на Cloud Run, AWS Lambda или Cloudflare Workers.

Из ограничений: эндпоинт должен быть доступен по HTTPS с валидным сертификатом, IP-allowlist Google публикует отдельно для тех, кому нужно ограничить входящие соединения. Скорость доставки — до сотен событий в секунду на одну подписку, ограничения уточняются в документации.

Webhooks доступны в публичной превью для всех проектов Gemini API, без отдельного тарифа. Поддержка распространяется и на студию AI Studio: можно зарегистрировать тестовый эндпоинт через ngrok или RequestBin прямо из интерфейса. Документация содержит примеры на Python, Node.js и Go с проверкой подписи.
