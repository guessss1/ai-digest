22 апреля Google объявил общую доступность Gemini Embedding 2. Модель переехала из preview в GA и теперь доступна как через Gemini API, так и через Vertex AI. Главная особенность — нативная мультимодальность: одна модель, одно единое векторное пространство, в которое кодируются тексты, изображения, видео и аудио.

Что это даёт на практике. Раньше, чтобы сделать поиск по смешанному контенту (например, найти видео по текстовому запросу или изображение по описанию из аудио), приходилось держать несколько эмбеддинг-моделей и либо сводить их через cross-encoder, либо строить прослойки-переводчики. С Gemini Embedding 2 любой запрос любой модальности сравнивается с любым документом любой модальности по обычному косинусному расстоянию.

По цифрам. На MTEB (Multilingual) — 69.9, на MTEB Code — 84.0. MTEB Multilingual — стандартный бенчмарк для оценки эмбеддингов в текстовых задачах на многих языках, 69.9 ставит модель в верхнюю часть таблицы. MTEB Code — более узкий бенчмарк для поиска по коду, 84.0 — высокий результат для универсальной, а не специализированной на коде модели.

Для разработчиков это значит несколько конкретных вещей. Если у вас поиск в продукте, где пользователи работают с разнотипным контентом (документы + скриншоты + аудиозаписи встреч + короткие видео), теперь можно убрать фрагментированный пайплайн и индексировать всё в одной коллекции. Если у вас RAG на кодовой базе — Gemini Embedding 2 достаточно силён на коде, чтобы не тащить отдельную модель вроде CodeBERT.

Для продакшна релевантны детали, которые Google пока не раскрыла подробно в публичном посте: точная размерность эмбеддинга, поддержка Matryoshka-представлений (укороченных векторов для экономии места), цена за 1K входных токенов/изображений, лимиты частоты. Эти параметры обычно указываются в документации API — их стоит проверить перед тем, как оценивать стоимость миграции с текущих эмбеддингов.

Доступ через Vertex AI важен для команд в GCP-экосистеме: интеграция с Matching Engine, BigQuery-векторными индексами и IAM — всё уже есть, не нужно поднимать отдельную инфраструктуру.

GA вместо preview означает SLA и стабильный контракт API. Если до этого вы откладывали интеграцию из-за preview-статуса — сейчас подходящий момент посмотреть, как Gemini Embedding 2 заменит ваш текущий стек (text-embedding-3, Cohere, BGE и прочее), особенно если в продукте есть мультимодальный поиск.
