Mistral выкатила Medium 3.5 — плотную модель на 128 миллиардов параметров с контекстным окном на 256 тысяч токенов. По SWE-Bench Verified она показывает 77.6%, что ставит её в один ряд с топовыми проприетарными конкурентами на задачах исправления багов в реальных репозиториях. Главное отличие от прошлых релизов — переключаемые усилия рассуждения.

Разработчик задаёт уровень reasoning отдельным параметром запроса. На низком уровне модель отвечает быстро и подходит для чата или коротких подсказок. На высоком — тратит больше токенов на цепочку размышлений и используется для агентных запусков, где важна точность планирования. Подход напоминает то, как OpenAI и Anthropic делят модели на «быстрые» и «думающие», только здесь это один артефакт с настраиваемым поведением.

Параллельно команда обновила Vibe — облачный coding-агент Mistral. Раньше Vibe работал как ассистент в IDE и требовал постоянного присутствия разработчика: тот формулировал задачу, агент выполнял шаги, человек проверял диффы. Новая версия добавляет удалённые сессии. Можно запустить несколько агентов параллельно на разных задачах — например, один чинит CI, второй пишет миграцию, третий обновляет зависимости. Сессии живут на стороне Mistral и не привязаны к локальной машине.

Что это даёт на практике:

- Долгие задачи (рефакторинг, апгрейды фреймворков) не блокируют редактор.
- Несколько агентов работают над разными ветками одного репозитория одновременно.
- Результаты приходят как готовые pull request-ы, которые ревьюит человек.

Vibe использует Medium 3.5 в качестве основной модели и опирается на её увеличенный контекст: 256k токенов хватает, чтобы держать в памяти десятки файлов проекта без агрессивного RAG. Конфигурируемый reasoning важен здесь напрямую — на этапе планирования агент включает высокий уровень, на этапе генерации кода переключается на стандартный. Это снижает счёт за токены, не теряя в качестве сложных шагов.

Medium 3.5 доступна через La Plateforme, AWS Bedrock и партнёрские облака. Vibe пока распространяется через раннюю программу, расширения для VS Code и JetBrains обновлены до версии с удалёнными сессиями. Mistral заявляет совместимость API с предыдущей Medium-серией, так что миграция требует только смены идентификатора модели.

Релиз вписывается в общую тенденцию: вендоры всё чаще продают не голую модель, а связку «модель плюс агентский раннер». Для команд, которые уже строят пайплайны на Mistral, апдейт даёт два рычага сразу — больше токенов в контексте и готовый control plane для длинных coding-задач.
