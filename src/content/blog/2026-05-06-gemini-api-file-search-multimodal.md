Google обновила File Search — встроенный в Gemini API сервис семантического поиска по корпусам файлов. До сих пор он индексировал только текст. Теперь поддерживаются изображения и документы со смешанным содержимым: PDF со схемами, скриншоты интерфейсов, отсканированные страницы с таблицами. За индексацию отвечает Gemini Embedding 2 — мультимодальная модель эмбеддингов, выпущенная Google ранее в этом году.

Главная цель — упростить RAG для тех, кто не хочет поднимать отдельную векторную базу. Pinecone, Weaviate, pgvector и аналоги по-прежнему живут, но для проектов на Google Cloud File Search закрывает базовый сценарий из коробки. Загружаешь файл — он автоматически делится на чанки, каждый чанк превращается в эмбеддинг и попадает в управляемый индекс. Поиск выполняется обычным вызовом generate_content с указанием корпуса.

Что добавилось в обновлении:

- Поддержка изображений: PNG, JPEG, WebP индексируются как самостоятельные единицы.
- Документы со смешанным содержимым: PDF с картинками теперь возвращают релевантные страницы, а не только текстовые фрагменты.
- Метаданные: каждому файлу можно навесить произвольные пары ключ-значение и фильтровать поиск по ним.
- Постраничные цитаты: ответ модели содержит ссылки на конкретные страницы исходных документов.

Метаданные критичны для практических корпусов. Один индекс может хранить документы разных проектов, языков, отделов; запрос фильтруется по `project_id`, `language` или `access_level` без перестройки индекса. Раньше для этого приходилось городить теги поверх внешней базы.

Постраничные цитаты упрощают аудит. Когда модель отвечает «по данным вашего PDF, цена контракта 120 тысяч», ответ содержит структурированную ссылку на страницу 7 файла X. Это базовое требование для использования в финансах, юриспруденции и комплаенсе, где галлюцинации недопустимы и каждое утверждение должно быть проверяемым.

Связка с Batch API даёт ещё один сценарий — массовое аналитическое прогон по корпусу. Вы готовите тысячу запросов, каждый указывает на тот же File Search-корпус с фильтрами. Batch API запускает их асинхронно по сниженной ставке, результаты складываются в JSONL. Это покрывает паттерн «прогнать одну анкету по тысяче резюме» или «извлечь ключевые поля из всех договоров за квартал» без написания собственного оркестратора.

Pricing остаётся прежним: оплачивается индексация (за токены документов), хранение (за гигабайты в месяц) и сами запросы. Лимиты на размер корпуса подняты, но точные цифры зависят от тарифа. Для тех, кто уже использовал текстовый File Search, обновление обратно совместимо — старые корпусы продолжают работать, переиндексация для добавления визуальной модальности — отдельная операция через API.

Документация и примеры доступны в Gemini API reference; SDK для Python и Node.js обновлены до версий с поддержкой новых параметров.
