OpenAI запустила Workspace Agents — облачные агенты внутри ChatGPT для многошаговых рабочих задач. В отличие от обычного чата, агент продолжает выполнение после закрытия вкладки и возвращается с результатом, когда задача завершена.

Каждый агент работает в собственной виртуальной машине с полноценным браузером, файловой системой и Python-окружением. Из коробки доступны коннекторы к Slack, Salesforce, Google Workspace, Notion, Jira, GitHub и Microsoft 365. Подключение оформляется через OAuth на стороне администратора рабочего пространства.

Сценарии, которые OpenAI приводит в анонсе, типичны для офисной автоматизации:

- Сбор еженедельного отчёта по продажам из Salesforce и публикация в Slack.
- Триаж входящих писем и создание задач в Jira.
- Подготовка черновиков ответов клиентам с подтягиванием истории из CRM.
- Регулярный аудит уязвимостей по списку репозиториев.

Агенты создаются либо через UI с визуальным редактором, либо описанием на естественном языке. Второй путь напоминает Custom GPTs: пишешь в диалоге, что должен делать агент, ChatGPT генерирует конфигурацию, после чего её можно править вручную.

Workspace Agents идут на смену Custom GPTs для тарифов ChatGPT Business, Enterprise и Edu. До 6 мая использование агентов бесплатное и не считается в лимит сообщений. После 6 мая каждый запуск тарифицируется отдельной строкой по числу шагов и времени выполнения.

Custom GPTs остаются доступными для индивидуальных пользователей Plus и Pro. Миграция автоматическая: существующие GPT можно одним кликом превратить в Workspace Agent, инструкции и файлы переносятся, но коннекторы нужно подключать заново.

Для разработчиков важно: появился отдельный API `agents.workspace`, через который можно запускать агентов программно и получать результаты вебхуками. Под капотом это та же модель GPT-5.5 с расширенным tool use, но с управлением состоянием на стороне OpenAI. Длительность одной сессии — до 24 часов; для более долгих задач предусмотрен механизм снапшотов.

Безопасность строится на трёх уровнях. Администратор рабочего пространства задаёт список разрешённых коннекторов и доменов. Каждый агент при первом запуске проходит проверку instruction injection через отдельный классификатор. Действия с побочными эффектами — отправка писем, изменение записей в CRM — по умолчанию требуют подтверждения пользователя.

Из ограничений: пока нет поддержки Microsoft Teams и Zendesk, обещают до конца квартала. Также агенты не работают с локальными файлами на машине пользователя — только с файлами, загруженными в ChatGPT или доступными через коннекторы.
